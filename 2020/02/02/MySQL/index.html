2. 操作数据库

2.1 数据类型

  • 数值
    • tinyint 十分小的数据 1个字节
    • smallint 较小的数据 2个字节
    • mediumint 中等大小的数据 3个字节
    • int 标准的整数 4个字节
    • bigint 较大的数据 8个字节
    • float 浮点数 4个字节
    • double 浮点数 8个字节
    • decimal 字符串形式的浮点数 金融计算时候
  • 字符串
    • char 字符串固定大小的 0-255
    • varcahr 可变字符串 0-65535
    • tinytext 微型文本 2^8-1
    • text 文本串 2^16-1
  • 时间日期
    • date 日期 YYYY-MM-DD
    • time 时间 HH: mm:ss
    • datetime YYYY-MM-DD HH: mm:ss
    • timestamp 时间戳， 1970.1.1到现在的毫秒数
    • year 年份
  • null
    • 不要使用null进行运算，结果为NULL

2.2 字段属性

  • Unsigned：无符号的整数，不能声明为负数
  • Zerofill: 0填充的，不足的位数使用0来填充
  • 自增Auto_increment：通常用来设置唯一的主键，必须是整数类型；可以设置增量步长和起始值
  • 非空：NULL 和NOT NULL

6. 事务

6.1 什么是事务

  • 要么都成功，要么都失败。

6.2 ACID四大原则

6.2.1 原子性Atomic

  • 要么一起成功，要么一起失败。

6.2.2 一致性Consistent

  • 倘若事务操作失败，则回滚事务时，与原始状态一致。
  • 事务前后数据完整性保持一致。

####6.2.3 持久性 Durable

  • 事务结束后数据不随着外界原因导致数据丢失

####6.2.4 隔离性 Isolation

  • 当你在进行事务操作时，其他事务的操作不能影响到你的事务操作

7. 索引

7.1 索引的分类

  • 主键索引 primary key
    • 唯一标识，不可重复，只能有一个列作为主键
  • 唯一索引 unique key
    • 避免重复的列，唯一索引可以重复，多个列都可以标识为 唯一索引
  • 常规索引 key/index
    • 默认的
  • 全文索引 FullText
    • 在特定的数据库引擎下才有，MyISAM
    • 快速定位数据

7.3 索引原则

  • 索引是不是越多越好
  • 不要对进程变动数据加索引
  • 小数据量不需要加索引
  • 索引一般加在常用来查询的字段上

9. 数据库的规约，三大范式

9.1 设计规范

  • 不要用驼峰命名，用下划线

9.2 三大范式

9.2.1 第一范式 1NF

  • 要求数据库表的每一列都是不可分割的原子数据项。

9.2.2 第二范式 2NF

  • 前提：满足1NF
  • 每张表只描述一件事

9.2.3 第三范式 3NF

  • 前提：满足1NF和2NF
  • 确保数据表中的每一列数据都和主键直接相关，而不能间接相关
