面向对象

发表于 2020-02-05 | 分类于 设计模式

1. 面向对象

  • Object-Oriented Programming
  • 本质：以类的方式组织代码，以对象的组织（封装）数据
  • 抽象
  • 三大特性：
    • 封装
    • 继承
    • 多态​

2.类和对象

2.1 类

  • 抽象的数据类型
  • 使用new关键字创建对象

2.2 对象

  • 对象概念的具体实例

2.3构造器（方法）

  • 在创建对象时候调用，和类名相同，没有返回值；

    1.无参构造器可以省略

    2.调用有参构造器，无参构造器不执行

2.4 类图

3. 三大特性

3.1 封装

  • 属性私有：get，set

    高内聚：类的内部数据操作细节自己完成，不需要外部干涉。

    低耦合：尽量暴露少量的方法给外部使用。

    好处：提高程序的安全性，保护数据。

    ​ 隐藏代码的实现细节。

3.2 继承

  • 本质是对某一批类的抽象；
  • java里面是单继承；
  • Object类是所有类的父类；

3.2.1 super和this

super指向父类

this 指向本身

【super】：

  • 调用父类的构造方法，必须在构造方法第一个；
  • 必须只能出现在子类的方法和构造方法中；
  • super和this不能同时调用构造方法；

【this】：

  • this没有继承也可以使用，super只能在继承条件下使用。

3.3 多态

3.3.1 重写

不能被重写的方法：

  • static
  • final
  • private

3.3.2多态

  • 【父类的引用指向子类】：Person s1=new Student();
    • 对象能执行的方法：主要看对象左边的Person类型，与右边的Student关系不大。
    • 不能调用子类独有的方法。
  • 多态是【方法的多态】，属性没有多态；
  • 多态存在的条件：继承关系，方法需要重写，父类的引用指向子类对象

3.3.3 instanceof

  • instanceof 判断类型

3.3.4 类型转换

  1. 低转高：不需要强制转换

Person s1=new Student();

  1. 高转低：强制转换

Student student=(Student)s1;

4. static

4.1 静态属性

通过类.属性调用，不需要实例。

4.2 静态方法

  • 非静态的方法，可以访问静态方法。
  • 静态方法可以调用静态方法，但是不能访问非静态方法。

4.3 静态代码块

static只执行一次

5. 抽象类和接口

5.1 抽象类abstract

  • 抽象类与抽象方法：
    • 抽象方法不需要写内容public abstract void shout();
    • 抽象类里面可以有抽象方法和普通方法
    • 子类要实现抽象类的所有方法，除非子类也是抽象的。

5.2 接口 interface

  • 接口也是约束，约束与实现分离。
  • 接口可以实现implements伪多继承。
  • 接口中的方法都是抽象的public abstract,必须实现里面所有方法。

6. 内部类

  • 定义：在一个类中再定义一个类。
  • 内部类可以访问外部类的私有属性
  • 内部类中不能由static属性

6.1 成员内部类

6.2 静态内部类

  • 一旦加上static后，内部类就拿不到外部类的私有属性了。
  • 静态内部类的创建不依赖外部类

6.3 匿名内部类

  • 匿名内部类必须继承一个抽象类或者实现一个接口。
  • 匿名内部类没有类名，因此没有构造方法。

6.4 局部（方法）内部类

  • 写在方法里面
  • 不允许使用访问权限修饰符（public、private、protected）均不允许。
